# 竞品后台功能地图

来源：登录 `https://hm2-admin.hnqiliang.cn/` 后只读分析菜单、路由、页面文案和公开前端资源。目标是学习功能闭环，不学习前端样式。

## 角色体系

竞品后台存在 5 类角色：

- 管理员：全量配置、内容、投放、支付、系统账号。
- 商务：看数据、管理商务下的组长和代理、处理投诉反馈。
- 代理：创建推广、看推广链接、看订单和用户数据。
- 代理组长：管理代理，创建推广，查看组内数据。
- 编辑：编辑书籍、剧集、壁纸、H5，管理分类和上传内容。

可学习点：后台菜单不是单一管理员视角，而是按“管理员 / 商务 / 组长 / 代理 / 编辑”拆权限。我们已补后台账号角色化和投放组织绑定，商务/组长/代理会按数据范围裁剪。

## 功能模块

### 数据统计

- 用户统计：用户增长、注册、加桌、渠道/应用维度。
- 充值统计：充值金额、充值次数、支付方式、订单状态。
- 新增充值：新用户充值、纯新增充值、充值时空统计。
- 回收趋势：T+N 回收、投流成本、付费率、回收率。
- 作品排行：小说/短剧作品付费与转化排行。

我们已补：用户增长、回收趋势、充值时段、支付成功率、内容回收排行、内容转化漏斗、播放统计；数据统计页支持按今日、昨日、近 7 天、近 30 天、近 90 天、本月、上月和自定义日期做时间窗口统计，并展示当前周期与上一等长周期的收入、成本、付费订单和回本率对比；也支持按应用、商务、组长、代理、推广入口、推广码、平台、渠道、广告 ID、素材 ID 做交叉筛选，用户增长、回收趋势、充值时段、支付成功率、播放统计、内容转化和投放异常会按同一口径收窄；用户增长页按自然日和来源聚合新增用户、访问、注册、加桌、激活、付费用户、付费订单、净收入、成本、付费率、注册率、加桌率、激活率、回本率、新增成本、加桌成本，并按应用、平台、渠道、广告、素材和推广入口拆分来源；充值时段页按支付小时聚合订单、付费用户、净收入、退款、均单价、峰值小时和支付方式分布；支付成功率页按订单状态、支付通道和支付方式拆分成功率、失败率、待支付、退款率和净收入；播放统计页统一短剧播放和小说阅读，按内容、分集/章节和自然日聚合浏览/播放、观看用户、观看进度、完播、锁定曝光、下单和解锁；小时洞察按 24 小时聚合访问、浏览、锁定曝光、创建订单、支付订单、收入、成本、转化率和回本率，并自动给出峰值小时、内容承接、素材异常和投放动作建议；新增 `analytics_review_tasks` 复盘任务闭环，自动复盘建议可一键转待办，记录处理状态、负责人、截止时间、逾期/今日到期、处理人、备注、提醒次数和处理日志，关联推广入口时可直接更新预算保护线或暂停推广并衔接平台停投任务，任务可向负责人发送后台站内提醒；复盘任务已支持素材替换审批，记录原广告/创意/素材、新方案、提交人、审批人、审批备注和应用时间，审批通过后可直接应用到推广入口，并按基线/最新/变化快照追踪收入、成本、订单、转化率和回本变化；统计筛选也可保存为常用方案，支持本人私有、共享给团队、快速套用和按权限删除。

后续可补：素材替换自动推送到外部投放平台、复盘效果按负责人绩效统计。

- 订单批量回传：订单管理、已支付订单、退款订单等筛选页新增“批量回传当前页已支付订单”，自动排除测试/未支付订单，复用单笔手动回传和角色数据范围校验，适合投放平台漏回传后的批量补发。
- 回传日志分析：回传配置页新增按状态、事件、订单、推广码、平台、应用、广告和素材筛选；展示成功/失败/跳过汇总和失败原因 Top5，并支持导出当前筛选回传 CSV。
- 投放异常监控：数据统计新增投放异常看板，自动聚合失败/不可重试回传、低转化素材、高退款素材、低回收推广链接和自动停投入口，沿用后台角色数据范围裁剪。
- 站内预警待办：投放异常可一键同步为站内预警通知，按稳定指纹去重并记录出现次数、最后发现时间、处理状态和处理备注；商务/组长/代理仍按投放范围处理可见预警。
- 预警外部通知：投放异常页新增预警多通道配置、模板、最低优先级、可发送状态、失败重试开关、紧急升级渠道和失败升级策略；支持全局或多个接收人配置 Webhook、企业微信机器人、邮箱、手机号和站内渠道，按全局、角色、指定商务/组长/代理范围和预警类型分发；单条/批量发送会按“接收人 + 渠道”拆分目标，Webhook/企微沿用 mock/HTTP 能力，企业微信机器人支持官方 timestamp/sign 加签，邮件可走 SMTP 真实发送并写入邮件台账，短信可走自定义 HTTP 供应商接口并按响应写入预警流水，供应商短信回执可通过 API 验签后更新预警短信成功/失败状态，站内渠道会真正生成后台站内消息并同步写入预警通知流水；发送失败达到配置阈值后可自动加发指定升级渠道，并在 `operation_alert_notification_logs` 按渠道独立去重、记录升级原因和留痕。
- 平台停投任务：投放异常页新增停投任务看板和平台分布，自动停投入口会生成任务；任务支持模拟成功/失败、HTTP 接口执行、平台适配器执行、批量执行、人工完成和取消，并按商务/组长/代理数据范围裁剪可见与可执行范围；平台适配器可按巨量/快手/百度等模板维护别名、执行接口、查询接口、鉴权、执行字段映射、查询字段映射、授权账号档案和响应规则，支持测试连接并记录最近测试状态、请求和响应；授权档案可按应用、渠道、代理、广告和平台别名匹配不同 token/接口，支持 token 到期时间、refresh token、刷新接口、自动提前刷新和手动刷新，任务会记录命中的适配器和授权账号，并回写平台请求单号、平台状态、平台编码和平台消息；平台返回处理中状态时任务保留 processing，可单条或批量回查平台最终结果；平台返回授权失败或限流时会生成专门预警，限流和可重试失败按退避时间再执行，授权失败会阻断批量重试直到账号刷新或人工处理。
- MySQL 索引增强：订单、推广链接、推广事件、投放消耗、内容事件、回传日志、广告平台、广告分层策略、广告位、广告事件、小程序配置和同步任务索引表新增联合索引，便于后续切 MySQL 后做广告/素材/小程序维度查询。
